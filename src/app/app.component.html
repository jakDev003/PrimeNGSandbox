Description of the issue:

The purpose of this demonstration is simply to show that when there is a 'p-dialog' open on a page that has a scrollbar, PrimeNG will add a class called 'p-overflow-hidden' which will block scrolling on the parent page. This is great. The issue arises when there is a confirmation Dialog on top of it. Once the confirmation Dialog is closed PrimeNG will then remove the class of 'p-overflow-hidden' but the original 'p-dialog' is still open so the parent page should not be scrollable.

Steps To Reproduce:

  1. Notice there is a scrollbar in the window of the parent page.
  2. Click on the 'Show' button.
  3. The 'p-dialog' should now open. Notice the parent page is now blocked from scrolling and the scrollbar is gone, and in the developer tools notice the 'body' element now has a class of 'p-overflow-hidden'
  4. Click on the 'Confirm Dialog' button.
  5. The 'p-confirmDialog' should now be open, notice the 'body' element still has a class of 'p-overflow-hidden'
  6. Click on either 'No' or 'Yes' button to close the 'p-confirmDialog'.
  7. The 'p-confirmDialog' should now be closed. Notice the 'body' element now has the class of 'p-overflow-hidden' removed and the scrollbar appears and is functional on the parent page.
